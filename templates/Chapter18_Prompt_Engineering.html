Chapter 18: Prompt Engineering

1. Introduction to Prompt Engineering

Prompt Engineering is the practice of designing, refining, and optimising the instructions given to AI language models so they produce accurate, useful, and relevant responses. It is a critical skill for anyone working with modern AI systems such as ChatGPT, Claude, Gemini, or any LLM-powered application.

A prompt acts as a communication bridge between humans and AI systems. The quality of the output depends heavily on the quality of the prompt — a well-structured prompt can dramatically improve response accuracy, tone, and usefulness.

Core principle: Better Prompt → Better Response. The same model will produce very different results depending on how it is instructed.

Where Prompt Engineering is Used

Chatbots & Assistants

Customer support bots, HR assistants, and virtual agents all rely on carefully crafted system prompts to stay on-topic and on-brand.

Code Generation

Tools like GitHub Copilot and Cursor use prompts to generate, explain, debug, and refactor code across languages.

Content Creation

Marketing teams use prompt templates to produce blog posts, social media captions, and ad copy at scale.

Data Extraction

Structured prompts instruct models to extract named entities, sentiment, or key facts from unstructured text.

Education

Personalised tutoring systems use prompts to adapt explanations to a student's age, subject, and skill level.

Research & Analysis

Analysts prompt LLMs to summarise documents, compare sources, and generate literature review drafts.

2. Why Prompt Engineering Matters

The same underlying model can give dramatically different quality outputs depending on how it is prompted. Prompt engineering bridges the gap between a model's raw capability and what it actually produces in practice.

❌ Weak Prompt

Write about AI.

Too vague. The model must guess the audience, length, tone, and purpose. Output is generic and often unusable.

✅ Strong Prompt

Write a 300-word introduction to Artificial Intelligence for first-year engineering students. Use simple language, include a real-world example, and end with why it matters for their careers.

Specific audience, length, tone, content requirement, and closing instruction. Output is targeted and immediately usable.

Benefits of Good Prompt Engineering

BenefitExplanation
Improves response qualitySpecific instructions guide the model toward accurate, relevant output
Reduces ambiguityClear context leaves less room for misinterpretation
Generates consistent outputsReusable templates produce reliable results across many runs
Saves time and effortGood prompts reduce back-and-forth iteration cycles
Enhances productivityAutomating repetitive tasks with reliable prompts frees human effort
Reduces hallucinationsGrounded, contextual prompts lower the chance of fabricated facts

3. Anatomy of a Good Prompt

Every effective prompt is built from a set of core components. Not all components are required every time, but knowing each one helps you construct prompts that leave no room for misinterpretation.

ComponentPurposeExample
TaskWhat should the AI do?"Summarise the following article"
ContextBackground information the model needs"The article is from a medical journal targeting general readers"
ConstraintsRules, limits, or things to avoid"In under 150 words. Do not use jargon."
Output FormatHow the response should be structured"Return as 3 bullet points"
AudienceWho will read or use the output"For a non-technical executive audience"
Tone / StyleDesired voice or register"Professional but approachable"
ExamplesDemonstrations of expected outputOne-shot or few-shot examples (see sections 5–6)

Full Example Using All Components

Task:        Explain Machine Learning

Context:     For a workshop aimed at business managers who have no technical background

Audience:    Non-technical professionals

Length:      300 words maximum

Format:      Bullet points, each starting with a bold keyword

Tone:        Friendly and encouraging

Avoid:       Mathematical formulas, code snippets

4. Zero-Shot Prompting

In zero-shot prompting, the AI is asked to perform a task with no examples provided. The model relies entirely on its pre-trained knowledge and the clarity of the instruction. This works well for tasks the model has seen many times during training.

Translate the following sentence into Tamil:
"Artificial Intelligence is transforming industries."

✅ Zero-Shot — Classification

Classify the following review as Positive, Negative, or Neutral:

"The delivery was fast but the product quality was disappointing."

✅ Zero-Shot — Summarisation

Summarise the following paragraph in one sentence:

"Large language models are trained on massive datasets..."
When to use Zero-Shot: Simple, well-defined tasks like translation, classification, summarisation, or factual Q&A. If the output quality is low, add examples (one-shot or few-shot).

Zero-Shot vs. Few-Shot Performance

Task TypeZero-Shot ReliabilityNotes
TranslationHighLLMs are extensively trained on multilingual data
Sentiment ClassificationHighWorks well with clear categories
Custom Format OutputLow–MediumExamples help the model understand the exact format needed
Domain-Specific ReasoningLowFew-shot or RAG recommended

5. One-Shot Prompting

One-shot prompting provides a single example of the input-output pair before asking the model to perform the task. The example anchors the model's understanding of the expected format, tone, and style.

Example:
Input:  Hello
Output: Vanakkam

Now translate:
Input:  Good Morning
Output:
Why one example helps: The model doesn't have to guess what "translate to Tamil" means in your specific context. The example shows the desired format (no punctuation, casual register, etc.) which would be hard to specify in words alone.

One-Shot for Formatting Tasks

Example:
Input:  "The product arrived broken."
Output: {"sentiment": "Negative", "issue": "Product damage", "priority": "High"}

Now process:
Input:  "Delivery was on time and the item looks great!"
Output:

6. Few-Shot Prompting

Few-shot prompting provides multiple (typically 3–5) input-output examples before the actual task. This gives the model a clearer pattern to follow and significantly improves accuracy on tasks that require a specific structure, vocabulary, or classification scheme.

Input: Cat      → Output: Animal
Input: Rose     → Output: Flower
Input: Eagle    → Output: Bird
Input: Salmon   → Output: Fish

Input: Mango    → Output:

Few-shot prompting often improves accuracy significantly, especially for custom classification, data extraction, and domain-specific generation tasks.

How Many Examples?

3–5 examples are usually enough. More examples consume context window tokens and can confuse the model if they are inconsistent.

Example Quality Matters

Diverse, representative examples outperform repetitive ones. Cover edge cases and different sub-categories in your examples.

Order Matters

Research shows the last example in the list has the most influence. Place your most representative example last.

Few-Shot for Sentiment + Reason

Input: "The app crashes every time I open it."
Output: Sentiment: Negative | Reason: Stability issue

Input: "Great battery life, very impressed."
Output: Sentiment: Positive | Reason: Hardware quality

Input: "It works fine but the UI is confusing."
Output: Sentiment: Mixed | Reason: Usability concern

Input: "Absolutely love the new update!"
Output:

7. Chain of Thought (CoT) Prompting

Chain of Thought prompting instructs the model to reason step by step before giving a final answer. This dramatically improves performance on tasks that require multi-step reasoning — mathematics, logic puzzles, code debugging, and cause-effect analysis.

Solve the following problem step by step:

If a train travels 60 km in 1 hour,
how far will it travel in 5 hours?

Think through each step before giving the final answer.

Why CoT Works

Forces Intermediate Steps

The model must generate intermediate tokens for each reasoning step, catching errors that direct answering would skip.

Improves Accuracy on Math

GPT-4 and Claude solve significantly more grade-school math problems correctly with CoT than without it.

Makes Reasoning Auditable

You can inspect each step to find where the model went wrong, making debugging much easier.

CoT Trigger Phrases

PhraseEffect
"Think step by step."General-purpose CoT trigger — works on most LLMs
"Let's reason through this carefully."Encourages slower, more deliberate reasoning
"Show your working."Useful for maths and logic
"Before answering, list the facts given."Grounds the model in provided information first
"What are the possible approaches? Then choose the best."Encourages exploration before committing to an answer

Zero-Shot CoT Example

Q: A farmer has 17 sheep. All but 9 die. How many are left?

A: Let's think step by step.
   "All but 9 die" means 9 sheep survive.
   Therefore, 9 sheep are left.
Zero-Shot CoT: Simply appending "Let's think step by step." to a prompt (without any example reasoning) is enough to trigger chain-of-thought behaviour in GPT-4 class models.

8. Role Prompting

Role prompting assigns a persona or professional identity to the model. This shapes vocabulary, depth of expertise, communication style, and the assumptions the model makes about the user.

You are a senior Python trainer with 10 years of experience
teaching beginners.

Explain functions in Python using a simple analogy and
a short code example. Avoid jargon.

How Role Affects Output

Role AssignedHow Output Changes
Senior Software ArchitectFocuses on design patterns, scalability, trade-offs
Primary School TeacherUses simple language, analogies, avoids technical terms
Financial AnalystEmphasises numbers, risk, ROI, regulatory considerations
Sceptical ReviewerChallenges assumptions, identifies weaknesses
Career CounsellorFocuses on goals, motivations, and actionable next steps
Devil's AdvocateArgues the opposite position to stress-test an idea

Advanced Role Prompt Example

You are an experienced UX researcher conducting a usability review.

Your job is to evaluate the following mobile app onboarding flow
and identify 3 usability issues with specific improvement suggestions.

Be critical and specific. Reference established UX heuristics where relevant.

[Paste onboarding flow description here]
Tip: Combine role prompting with context and constraints for best results. A role alone helps — a role with a specific task, audience, and output format is far more powerful.

9. Structured Outputs

Structured output prompting instructs the model to return its response in a machine-readable format. This is essential when the output will be parsed by code, stored in a database, or fed into another system.

JSON Output

Extract the following details from the text and return as JSON.
Do not add any explanation — return only the JSON object.

Text: "Hi, I'm Priya Sharma, 28 years old, from Bangalore."

Expected format:
{
  "name": "",
  "age": "",
  "city": ""
}

Supported Output Formats

FormatUse CaseExample Instruction
JSONAPIs, databases, code integration"Return only a valid JSON object."
XMLEnterprise systems, document interchange"Wrap the output in XML tags."
MarkdownDocumentation, wikis, README files"Format as a Markdown table."
CSVSpreadsheets, data pipelines"Return as comma-separated values with a header row."
HTMLWeb content, email templates"Return as an HTML unordered list."
YAMLConfig files, DevOps pipelines"Return as valid YAML."

Markdown Table Example

List the top 5 programming languages in 2024 with their
primary use case. Return as a Markdown table with columns:
Language | Primary Use Case | Popularity Rank
Best Practice: Always include "Return only the [format]. Do not add any explanation or preamble." — this prevents the model from wrapping the output in prose before the JSON or CSV.

10. Prompt Templates

Prompt templates are reusable prompt structures with placeholders that can be filled in for different inputs. They enforce consistency, reduce errors, and make it easy to automate prompt generation in applications.

General-Purpose Template

Your audience is [AUDIENCE].

Task: [TASK DESCRIPTION]

Constraints:
- Length: [WORD/CHARACTER LIMIT]
- Tone: [TONE]
- Avoid: [THINGS TO AVOID]

Output format: [FORMAT]

Length: [WORDS]. Tone: [TONE].
Include: [REQUIRED ELEMENTS].

11. AI Hallucinations

Hallucinations occur when an AI model generates information that sounds plausible but is factually incorrect, fabricated, or completely unsupported by its training data or the provided context. This is one of the most serious reliability challenges with LLMs.

Factual Hallucination

The model invents facts — fake citations, wrong dates, non-existent products, or incorrect statistics — presented with full confidence.

Contextual Hallucination

The model ignores or contradicts information provided in the prompt and generates something inconsistent with the given context.

Instruction Hallucination

The model fails to follow the explicit instructions in the prompt, producing output in the wrong format, length, or style.

Root Causes

CauseExplanation
Insufficient contextThe model fills knowledge gaps with plausible-sounding but fabricated information
Ambiguous promptsVague instructions leave the model too much creative freedom
Training data gapsTopics not well covered in training lead to guessing
Overconfidence biasModels are trained to give confident answers even when uncertain
Knowledge cutoffEvents after the training cutoff are not known to the model

Reduction Techniques

TechniqueHow It Helps
Provide more contextGrounding the model in specific facts reduces the need to fabricate
Ask for citations"Cite your source" forces the model to justify claims (verify externally)
Retrieval-Augmented Generation (RAG)Injects real, verified documents into the prompt at query time
Instruct to say "I don't know""If you are not certain, say 'I don't know' rather than guessing"
Temperature = 0Reduces randomness — model sticks to highest-probability tokens
Verify outputsHuman review or automated fact-checking pipelines catch errors

Anti-Hallucination Prompt Example

Answer the following question using only the information
provided below. If the answer is not in the text, respond
with "The provided text does not contain this information."
Do not use your general knowledge.

Text: [PASTE DOCUMENT HERE]

Question: [YOUR QUESTION]

12. Prompt Engineering Best Practices

1. Be Specific

Vague prompts produce vague answers. Include the topic, format, audience, length, and purpose in every prompt.

2. Provide Context

Tell the model what it needs to know. Include background, constraints, and the reason for the task.

3. Define the Audience

"For a 10-year-old" vs "for a PhD candidate" will produce completely different language and depth.

4. Specify Output Format

Tell the model exactly how to structure the output — JSON, bullet points, numbered steps, table, paragraph, etc.

5. Break Complex Tasks into Steps

Instead of one massive prompt, chain multiple simpler prompts. Each step's output feeds the next.

6. Use Examples

One-shot or few-shot examples are the fastest way to communicate a non-standard format or style requirement.

7. Iterate and Refine

Treat prompts like code — test, measure output quality, and improve. Keep a prompt log to track what works.

8. Assign a Role

Role prompting (e.g. "You are a senior data engineer") aligns the model's vocabulary, depth, and assumptions to your needs.

9. Tell It What Not to Do

Negative constraints ("Do not use jargon", "Do not repeat the question", "Avoid bullet points") are as important as positive ones.

10. Use Delimiters

Use triple quotes, XML tags, or dashes to clearly separate instructions from input content, preventing confusion.

Using Delimiters — Example

Summarise the text enclosed in triple quotes in 2 sentences.

"""
Large language models are neural networks trained on vast corpora
of text. They learn to predict the next token in a sequence,
which gives rise to emergent capabilities like reasoning,
translation, and code generation...
"""

13. Advanced Prompting Techniques

Beyond the core techniques, several advanced strategies help extract better performance from LLMs for complex or specialised tasks.

Self-Consistency Prompting

Generate the same prompt multiple times (with higher temperature), then take the majority-vote answer. Reduces the impact of single-run errors on reasoning tasks.

Solve this logic puzzle step by step. [PUZZLE]
(Run this prompt 5 times and select the most common answer.)

ReAct (Reason + Act)

The model alternates between reasoning steps and tool-use actions (web search, code execution, database queries). Used in agentic AI systems.

Thought: I need to find the current price of gold.
Action: search("current gold price per gram India 2024")
Observation: [search result]
Thought: Now I can calculate the cost.
Answer: ...

Retrieval-Augmented Generation (RAG)

Instead of relying on the model's memory, relevant documents are retrieved from a vector database and injected into the prompt context at query time. Dramatically reduces hallucinations for factual tasks.

Use only the following context to answer the question.

Context:
[Retrieved document chunks injected here automatically]

Question: [USER QUESTION]

Prompt Chaining

Break a complex task into a sequence of simpler prompts. The output of each step becomes the input to the next.

Step 1 → Extract key facts from the article
Step 2 → Identify the main argument using the key facts
Step 3 → Write a critique of the argument
Step 4 → Summarise the critique in 2 sentences

Tree of Thought (ToT)

The model explores multiple reasoning branches simultaneously, evaluates each, and selects the most promising path — like a search tree for problem solving.

Consider three different approaches to solve this problem.
For each approach, briefly explain the reasoning path.
Then select the best approach and solve it fully.

Problem: [YOUR PROBLEM]
Technique Selection Guide: Simple tasks → Zero-Shot Format-sensitive → Few-Shot Multi-step reasoning → CoT / ToT Factual / document Q&A → RAG Tool-using agents → ReAct

15. Mini Project

Build a Prompt Library — a structured collection of reusable, tested prompts organised by domain and technique.

Suggested stack: Python · JSON or SQLite for storage · Streamlit or Flask for a simple web UI · Optional: tag-based search and version tracking.

Project Steps

1

Define Categories

Organise prompts by domain: Content Creation, Code Generation, Customer Support, Education, Research, Data Extraction.

2

Design the Data Model

Each prompt entry should store: title, category, technique (zero-shot/CoT/etc.), template text, example output, tags, and version number.

3

Write and Test Prompts

Author at least 5 prompts per category. Test each on your target LLM and record output quality scores.

4

Build the UI

A simple Streamlit app with a search/filter panel, prompt preview, "Copy to Clipboard" button, and a form to add new prompts.

5

Add Version Control

Track changes to each prompt over time. Store previous versions so you can roll back if a new version underperforms.

6

Export & Share

Allow export to JSON or CSV. Optionally publish to a public GitHub repo or internal team wiki.
