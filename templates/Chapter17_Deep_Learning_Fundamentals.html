Chapter 17: Deep Learning Fundamentals

1. Introduction to Deep Learning

Deep Learning is a subset of Machine Learning that uses Artificial Neural Networks with multiple layers to automatically learn hierarchical representations from raw data. Unlike classical ML, no manual feature engineering is required — the network discovers features on its own given enough data.

Deep Learning powers modern technologies such as ChatGPT, image recognition systems, voice assistants, autonomous vehicles, recommendation engines, and medical diagnosis systems.

The AI Hierarchy: Artificial Intelligence  ⊃  Machine Learning  ⊃  Deep Learning

Key Application Areas

Language AI

ChatGPT, Claude, Gemini, Llama — all powered by Transformer-based deep learning models.

Computer Vision

Object detection, face recognition, and medical imaging rely on Convolutional Neural Networks.

Autonomous Systems

Self-driving vehicles use real-time deep learning to perceive and respond to their environment.

Medical Diagnosis

Models detect diseases from X-rays, MRIs, and pathology slides at near-radiologist accuracy.

2. Why Deep Learning?

Traditional machine learning requires domain experts to manually craft features (edges, frequencies, shapes). Deep learning removes this bottleneck — given sufficient data and compute, it discovers those features automatically and improves continuously as data grows.

CapabilityTraditional MLDeep Learning
Feature engineeringManualAutomatic
Performance on big dataPlateausKeeps improving
Unstructured data (images, text)LimitedExcellent
InterpretabilityHigherLower (black-box)
Training compute requiredLowHigh

When to use Deep Learning: Use it when you have large labelled datasets, high-dimensional inputs like images or long text, and sufficient compute. For small structured datasets, classical methods (gradient boosting, logistic regression) often perform better.

3. Artificial Neural Networks (ANN)

Artificial Neural Networks are inspired by the human brain. They consist of interconnected neurons organised into layers. Each neuron receives weighted inputs, adds a bias, applies a non-linear activation function, and passes its output to the next layer.

Input Layer       (raw features — pixel values, sensor readings, etc.)
      ↓
Hidden Layer 1    (detects low-level patterns — edges, tones)
      ↓
Hidden Layer 2    (combines patterns — shapes, phrases)
      ↓
Output Layer      (final prediction — class probability or value)

Each neuron receives input, performs calculations and passes information forward.

Shallow Network

1–2 hidden layers. Good for simpler tasks. Faster to train, lower memory cost.

Deep Network

3+ hidden layers. Learns abstract, hierarchical representations. Needs more data and compute.

Wide Network

Many neurons per layer. Increases model capacity but also memory and inference cost.

4. Perceptron

A perceptron is the simplest neural network unit — a single artificial neuron that maps a vector of inputs to a scalar output.

Output = Activation( Σ(wᵢ · xᵢ) + b )

Components

xᵢ — Inputs

Raw values fed into the neuron, e.g. pixel intensities, sensor readings, or word embeddings.

wᵢ — Weights

Learnable parameters. A large positive weight creates a strong excitatory connection to the next layer.

b — Bias

A learnable offset that shifts the activation threshold, giving the neuron additional flexibility.

Activation Function

A non-linear function (ReLU, Sigmoid, Tanh) applied to the weighted sum before producing output.

Output

The final value passed on to neurons in the next layer, or as the network's prediction.

Multi-Layer Perceptron (MLP): Stack many perceptrons in layers and you get an MLP — the classic fully-connected feedforward network. Adding depth lets the network compose simple functions into highly complex ones.

5. Activation Functions

Activation functions introduce non-linearity into neural networks. Without them, stacking multiple linear layers still produces a single linear function — depth would buy nothing.

FunctionFormulaRangeBest Used ForWeakness
Sigmoid1 / (1 + e⁻ˣ)(0, 1)Binary classification outputVanishing gradient for large |x|
Tanh(eˣ − e⁻ˣ) / (eˣ + e⁻ˣ)(−1, 1)Hidden layers in RNNsVanishing gradient
ReLUmax(0, x)[0, ∞)Hidden layers (default)Dying ReLU for x < 0
Leaky ReLUmax(0.01x, x)(−∞, ∞)Fixes dying ReLUSlight gradient noise
Softmaxeˣⁱ / Σ eˣʲ(0,1), sum = 1Multi-class output layerNumerically unstable at large values

ReLU

f(x) = max(0, x)

ReLU is computationally trivial (just a max operation), avoids vanishing gradients for positive inputs, and empirically trains faster than sigmoid/tanh. It has been the default choice for hidden layers since AlexNet (2012).

6. Forward Propagation

Forward propagation is the inference pass. Given input data, each layer transforms it in sequence until an output (prediction) is produced. No weights are updated during this stage.

Input x
      ↓
Layer 1:  z₁ = W₁·x + b₁,  a₁ = σ(z₁)
      ↓
Layer 2:  z₂ = W₂·a₁ + b₂, a₂ = σ(z₂)
      ↓
Output ŷ  (class probability or regression value)

Predictions are generated during this stage.

Linear Transform

z = Wx + b rotates and scales the data in weight space, projecting it into a new representation.

Activation

a = σ(z) applies the non-linearity, allowing the network to bend decision boundaries.

Prediction

The final layer outputs class probabilities via softmax (classification) or a raw value (regression).

7. Backpropagation

Backpropagation is the algorithm that trains a neural network. Using the chain rule of calculus, it computes how much each weight contributed to the error, then nudges weights in the direction that reduces the loss.

Forward Pass → Prediction ŷ
      ↓
Calculate Loss:  L(y, ŷ)
      ↓
Compute Gradients:  ∂L/∂W  via chain rule (backwards through layers)
      ↓
Update Weights:  W ← W − η · ∂L/∂W
      ↓
Repeat until convergence
W_new = W_old − learning_rate × gradient

Epoch vs Iteration: One epoch is a full pass over the training data. One iteration is a single weight-update step over a mini-batch. Training typically runs for tens to hundreds of epochs.

Learning Rate (η)

Controls the step size. Too large → the model diverges. Too small → very slow convergence. Typically 1e-3 for Adam.

Vanishing Gradient

Gradients shrink as they propagate through deep sigmoid layers, making early layers learn extremely slowly.

Exploding Gradient

Gradients grow exponentially through deep networks. Solved with gradient clipping or batch normalisation.

8. Loss Functions

Loss functions measure how far predictions are from actual values. It is the signal that backpropagation uses — minimising the loss is the entire objective of training.

Loss FunctionFormula (simplified)Use Case
Mean Squared Error (MSE)mean( (y − ŷ)² )Regression
Mean Absolute Error (MAE)mean( |y − ŷ| )Regression (robust to outliers)
Binary Cross-Entropy−( y log ŷ + (1−y) log(1−ŷ) )Binary Classification
Categorical Cross-Entropy−Σ yᵢ log ŷᵢMulti-Class Classification
Huber LossMSE for small errors, MAE for largeRegression + outlier tolerance

Cross-Entropy Intuition: If the model is confident and correct, cross-entropy is near zero. If the model is confident but wrong, the log term applies a very large penalty. This encourages well-calibrated probability outputs, not just correct labels.

9. Optimizers

Optimizers implement the rule for updating weights after gradients are computed. Modern optimizers converge faster and handle noisy gradients far better than vanilla gradient descent.

OptimizerDescriptionBest For
Gradient Descent (GD)Computes exact gradient over entire dataset. Slow for large data.Small datasets
SGD (Stochastic)Updates weights after each single example. Noisy but fast.Online learning
Mini-batch SGDUpdates on batches of 32–512 examples. Practical default.Most tasks
RMSPropScales learning rate by recent gradient magnitudes. Adaptive.RNNs, non-stationary problems
AdamCombines momentum + RMSProp. Per-parameter adaptive rates.Most tasks (default choice)
AdamWAdam with decoupled weight decay. Prevents over-regularisation.Transformer training (GPT, BERT)

Key Hyperparameters: Learning rate is the most important. Default for Adam: 1e-3. Batch size affects gradient noise — larger batches converge faster per epoch but may generalise slightly worse.

10. Convolutional Neural Networks (CNN)

CNNs are specialized neural networks that exploit the spatial structure of images. A small learnable filter (kernel) slides across the image computing dot products — detecting local patterns like edges, textures, and shapes regardless of where they appear.

Input Image  (H × W × C)
      ↓
Convolution + ReLU     (extract feature maps)
      ↓
Batch Normalisation    (stabilise training)
      ↓
Pooling Layer          (downsample, retain dominant features)
      ↓
[Repeat Conv blocks…]
      ↓
Flatten → Fully Connected
      ↓
Prediction (Softmax)

CNN Layer Types

Layer TypePurpose
ConvolutionalDetects spatial features using learnable kernels sliding over the input
Batch NormalisationStabilises training, reduces sensitivity to weight initialisation
Max PoolingReduces spatial dimensions while retaining the most prominent features
DropoutRandomly zeroes neurons during training to prevent overfitting
Fully ConnectedAggregates all spatial features for the final classification decision

Applications

  • Face Recognition — compares CNN embeddings to identify individuals
  • Medical Imaging — detects tumours, retinal disease, and fractures from scans
  • Object Detection — YOLO and Faster R-CNN locate and classify objects in real time
  • Image Classification — assigns a category label to an entire image
  • Image Generation — GANs and diffusion models use CNNs to synthesise photorealistic images
Landmark Architectures: AlexNet (2012) started the deep learning era. VGG, GoogLeNet, and ResNet followed. ResNet introduced skip connections (residual connections), allowing training of 100+ layer networks by solving the vanishing gradient problem.

11. RNN and LSTM

Standard feedforward networks treat each input independently. Recurrent Neural Networks (RNNs) add recurrent connections so the hidden state carries information across time steps — essential for sequences like text, audio, and sensor streams.

hₜ = tanh( Wₓ·xₜ  +  Wₕ·hₜ₋₁  +  b )

Applications

  • Language Translation — encoding source and decoding target sequences
  • Speech Recognition — processing audio frames in sequence
  • Text Prediction — predicting the next word or character
  • Time Series Forecasting — stock prices, weather, sensor data

LSTM networks overcome the memory limitations of standard RNNs.

Vanishing Gradient in RNNs: When backpropagating through many time steps, gradients get multiplied repeatedly and shrink to near zero. The network then fails to learn long-range dependencies — e.g. connecting a pronoun to a noun 50 words earlier.

RNN Variants

LSTM (Long Short-Term Memory)

Three gates — input, forget, output — control what to keep, discard, or expose from the cell state. Solves vanishing gradients over long sequences.

GRU (Gated Recurrent Unit)

A simpler variant of LSTM with fewer parameters (two gates). Often achieves comparable performance with faster training.

Bidirectional RNN

Processes the sequence in both directions (forward and backward), giving each time step access to full context.

12. Transformers

Transformers are modern deep learning architectures that revolutionised Natural Language Processing. Introduced in the 2017 paper "Attention Is All You Need", the Transformer replaced recurrence with self-attention — every token directly attends to every other token in a single pass.

Large Language Models such as GPT, Gemini, Claude and Llama are built on Transformer architectures.

Attention(Q, K, V) = softmax( Q·Kᵀ / √dₖ ) · V

Input Text
     ↓
Token Embeddings + Positional Encoding
     ↓
Multi-Head Self-Attention
     ↓
Feed-Forward Block  (×N layers)
     ↓
Generated Output

Key Components

Self-Attention

Each token builds a weighted combination of all other tokens, learning rich contextual relationships within the sequence.

Multi-Head Attention

Runs attention in parallel across multiple subspaces, capturing different types of relationships simultaneously.

Positional Encoding

Injects token position into embeddings — necessary because attention itself is permutation-invariant.

Layer Normalisation

Stabilises training inside residual blocks. Applied before attention (Pre-LN) in most modern models.

Advantages

  • Parallel Processing — unlike RNNs, all positions are computed simultaneously during training
  • Long Context Handling — long-range dependencies are trivial via direct attention
  • Scalable Training — scaling parameters and data reliably improves capability ("scaling laws")
  • State-of-the-Art Performance — dominates NLP, vision (ViT), audio, and multimodal tasks
Notable LLMs built on Transformers: GPT-4o Claude Gemini Llama 3 Mistral

14. Real-World Applications

Deep learning is embedded in the products and infrastructure most people use daily — often invisibly.

AI Assistants

ChatGPT, Claude, Gemini — Transformer LLMs that converse, reason, write code, and generate images.

Self-Driving Cars

Tesla Autopilot and Waymo fuse camera, radar, and LiDAR data via CNNs and Transformers for real-time perception.

Fraud Detection

Banks use deep learning to flag anomalous transactions in real time across millions of events per second.

Medical Diagnosis

Models detect diabetic retinopathy, skin cancer, and pulmonary embolism from images with radiologist-level accuracy.

Voice Recognition

Whisper, Siri, and Google Assistant use Transformer-based speech recognition and text-to-speech synthesis.

Image Generation

Stable Diffusion, DALL·E, Sora — diffusion models and video Transformers create images and video from text prompts.

Recommendation Systems

Netflix, Spotify, YouTube use deep collaborative filtering and sequence models to personalise feeds.

Protein Folding

AlphaFold 2 predicted the 3D structure of virtually every known protein — a breakthrough decades in the making.

15. Mini Project Idea

Build an Image Classification System that identifies whether an uploaded image contains a cat or a dog using a CNN model.

Suggested stack: Python · TensorFlow/Keras or PyTorch · Kaggle Dogs vs. Cats dataset · Transfer learning from a pre-trained ResNet or MobileNet · Flask or Streamlit for the web interface.

Project Steps

  1. Download and preprocess the Dogs vs. Cats dataset (resize to 224×224, normalise pixel values)
  2. Load a pre-trained CNN (e.g. MobileNetV2) and freeze its base layers
  3. Add a custom classification head (GlobalAveragePooling → Dense(128) → Dense(1, sigmoid))
  4. Train for 5–10 epochs with Adam and Binary Cross-Entropy loss
  5. Evaluate on the validation set; aim for >95% accuracy
  6. Build a simple web UI where users can upload an image and see the prediction
