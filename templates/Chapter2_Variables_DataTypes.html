Chapter 2: Variables, Data Types and Memory Management

1. Variables

A variable is a name used to store data in memory. Variables allow programs to work with dynamic values instead of fixed values.

name = "Suresh"
age = 35
salary = 50000

Python automatically determines the type of value assigned to a variable.

Unlike C or Java, Python does not require explicit datatype declarations.

Variable Naming Rules

  • Must begin with a letter or underscore (_)
  • Cannot begin with a number
  • Cannot contain spaces
  • Case sensitive
  • Keywords cannot be used as variable names

2. Dynamic Typing

Python is dynamically typed, meaning the datatype is determined during execution.

x = 100
print(type(x))

x = "Python"
print(type(x))

The same variable can hold different data types at different times.

3. Understanding Memory

x = 100

Python performs the following operations:

  1. Create an object 100 in memory.
  2. Create a reference variable x.
  3. Link x to the object.
x = 100
y = x

Both variables refer to the same memory object.

11. id() Function

The id() function returns the memory address of an object.

a = 100
b = 100

print(id(a))
print(id(b))

Often both IDs are the same because Python reuses objects.

12. Memory Management

Python automatically allocates and deallocates memory.

a = 100
a = 200

The old object becomes unused and may be removed later.

Reference Counting

Python tracks how many references point to an object.

13. Garbage Collection

Garbage Collection removes unused objects from memory.

import gc

gc.collect()

This helps prevent memory leaks.
